    系统的环境变量

    环境变量(Environment Variables) 是操作系统用于存储配置信息的一种方式，它们在操作系统的不同进程中传递特定的配置信息。环境变量通常用于存储系统路径、系统配置参数、用户信息等内容，供程序和脚本在运行时使用。

    环境变量

    为什么要配置环境变量：

    1. 简化操作：通过配置环境变量，可以避免在命令行或脚本中反复输入路径或配置信息。例如，当你安装某个软件后，通过将其路径加入环境变量中，你就可以在任何地方运行该软件，而无需每次都提供完整路径。
    2. 提高系统可移植性：通过环境变量配置系统相关的信息，软件在不同的系统上运行时无需修改代码，只需要调整相关的环境变量。
    3. 统一管理配置信息：环境变量可以集中存储系统和软件的配置信息，方便维护和管理。例如，不同的开发工具链（如Java、Python等）通常需要配置环境变量来找到其对应的库或工具。
    4. 系统与应用程序的通信：应用程序通过环境变量可以获取系统的一些全局信息，如操作系统类型、系统语言、临时目录位置等，帮助程序根据不同环境做出适应性调整。

    常见使用环境变量的例子：

    • **PATH**：最常见的环境变量之一，存储了系统可执行文件的搜索路径。配置PATH可以让你在命令行直接运行可执行文件而不需要输入其完整路径。
    • **JAVA_HOME**：指定Java安装目录的环境变量，便于各种Java开发工具找到Java安装路径。
    • **PYTHONPATH**：Python的搜索路径，通过配置它可以让Python知道在哪里寻找模块或包。

    总结来说，配置环境变量可以帮助我们简化系统管理和软件开发，减少重复配置的工作，并提高系统的灵活性和可维护性。

    PATH

    PATH 是操作系统中的一个环境变量，它指定了系统查找可执行文件的目录列表。当你在命令行或终端中输入一个命令时，操作系统会依次在 PATH 变量中定义的目录中查找该命令的可执行文件。如果找到了可执行文件，操作系统就会运行它；如果没有找到，则会报出“命令未找到”的错误。

    PATH就是系统维护一张表，里面写了可执行文件的相关路径

    PATH 的作用：

    • 简化命令执行：当你在命令行中输入一个程序或命令的名称时，你不需要提供完整的路径，只要这个程序所在的目录在 PATH 中，系统就会找到并执行该程序。例如，在终端中直接输入 python，系统会在 PATH 中定义的目录里搜索 python 可执行文件。
    • 系统自动查找命令：系统根据 PATH 中的目录顺序查找命令或程序的位置。因此，目录顺序也很重要，系统会首先在 PATH 中最先列出的目录中查找命令。

    PATH 的格式：

    在 Linux/macOS 中，PATH 变量中的目录是用冒号 : 分隔的。

    /usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin

    在 Windows 中，目录是用分号 ; 分隔的。

    C:\Windows\System32;C:\Windows;C:\Program Files\Go\bin

    修改 PATH 变量：

    1. **临时修改 PATH**：

      • Linux/macOS

        ：可以在命令行中直接修改：

        export PATH=$PATH:/new/directory/path

      • Windows

        ：可以通过命令提示符修改：

        set PATH=%PATH%;C:\new\directory\path

    2. **永久修改 PATH**：

      • Linux/macOS：编辑 ~/.bashrc 或 ~/.zshrc 文件，并添加 export PATH=$PATH:/new/directory/path，然后执行 source ~/.bashrc 来使其生效。
      • Windows：通过“系统属性” -> “高级系统设置” -> “环境变量”来永久修改 PATH。

    查看PATH

    使用python来查看系统的PATH

    import os

# 获取PATH环境变量
path_variable = os.environ.get('PATH')

# 打印PATH环境变量
print("系统的PATH环境变量是：")
print(path_variable)
